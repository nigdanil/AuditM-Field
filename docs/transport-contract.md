# AuditM-Field Transport Contract

AuditM-Field sends an exported inspection package to external systems through HTTP/Webhook storage adapters.

The recommended production path is:

```text
AuditM-Field PWA
  -> ZIP package
  -> HTTP upload / Webhook
  -> n8n or backend
  -> OCR / CV / LLM / RAG / storage
  -> external job result
```

## Request

Content type:

```text
multipart/form-data
```

Required fields:

| Field                  | Type        | Description                                   |
| ---------------------- | ----------- | --------------------------------------------- |
| `file`                 | File        | ZIP package generated by AuditM-Field         |
| `metadata`             | File / JSON | Full transport metadata as `metadata.json`    |
| `manifest`             | File / JSON | Export manifest as `manifest.json`            |
| `contractVersion`      | string      | Transport contract version, currently `1.0.0` |
| `packageFormatVersion` | string      | ZIP package format version, currently `1.0.0` |
| `inspectionId`         | string      | Inspection id                                 |
| `inspectionTitle`      | string      | Inspection title                              |
| `configId`             | string      | Config id                                     |
| `configName`           | string      | Config name                                   |
| `adapterId`            | string      | `http-upload` or `webhook`                    |
| `fileName`             | string      | ZIP filename                                  |
| `fileSize`             | string      | ZIP size in bytes                             |
| `exportedAt`           | string      | ISO datetime from manifest                    |

## metadata.json

```json
{
  "contractVersion": "1.0.0",
  "packageFormatVersion": "1.0.0",
  "adapterId": "webhook",
  "fileName": "auditm-field_abc_2026-05-10_10-00-00.zip",
  "fileSize": 123456,
  "inspectionId": "abc",
  "inspectionTitle": "Retail audit",
  "configId": "retail-shelf-audit",
  "configName": "Retail Shelf Audit",
  "exportedAt": "2026-05-10T10:00:00.000Z",
  "manifest": {}
}
```

## Response

The endpoint should return JSON:

```json
{
  "accepted": true,
  "jobId": "n8n-job-001",
  "message": "Package accepted for processing",
  "externalUrl": "https://n8n.example.com/execution/123",
  "status": "PROCESSING"
}
```

Field meanings:

| Field         | Type    | Description                                                   |
| ------------- | ------- | ------------------------------------------------------------- |
| `accepted`    | boolean | Whether package was accepted                                  |
| `jobId`       | string  | External job/execution id                                     |
| `message`     | string  | Human-readable message                                        |
| `externalUrl` | string  | Link to external job/execution                                |
| `status`      | string  | External status, for example `PROCESSING`, `ACCEPTED`, `DONE` |

## n8n setup

Recommended n8n workflow:

```text
Webhook node
  -> Read Binary File / Move Binary Data if needed
  -> Store ZIP to disk/S3/Drive/backend
  -> Parse metadata JSON
  -> Start OCR/CV/LLM pipeline
  -> Respond to Webhook
```

Webhook node settings:

```text
HTTP Method: POST
Response Mode: Respond to Webhook node
Binary data: enabled
Expected binary field: file
Metadata field: metadata
Manifest field: manifest
```

Respond to Webhook example:

```json
{
  "accepted": true,
  "jobId": "{{$execution.id}}",
  "message": "AuditM-Field package accepted",
  "externalUrl": "https://n8n.example.com/workflow/{{$workflow.id}}/executions/{{$execution.id}}",
  "status": "PROCESSING"
}
```

## Future response imports

Later AI/OCR suggestions should be imported back into AuditM-Field as a separate package or JSON file, preserving the source:

```json
{
  "source": "ai",
  "inspectionId": "abc",
  "photoId": "photo-001",
  "suggestions": []
}
```
