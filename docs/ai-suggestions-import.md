# AI suggestions import

AuditM-Field does not call AI providers directly from the PWA.

Recommended flow:

```text
AuditM-Field
  -> ZIP export
  -> n8n/backend
  -> OCR/CV/LLM/RAG processing
  -> ai-suggestions.json
  -> AuditM-Field import
  -> human review in Annotator
```

## Quick local test

Open:

```text
Export Center -> AI suggestions import
```

Click:

```text
Generate & import demo
```

Expected result:

```text
Demo AI suggestion generated and imported: 1/1
Open first affected photo
```

Open the affected photo in Annotator and check that the imported annotation has:

```text
source: ai
```

## AI review

In Annotator, AI suggestions can be reviewed by a human:

```text
Accept -> source changes from ai to human
Reject -> AI annotation is deleted
```

Available filters:

```text
All
Human
AI
Imported
```

AI suggestions also display:

```text
confidence
review status
source badge
```

## Import behavior

AuditM-Field validates:

```text
- JSON shape;
- source === "ai";
- inspectionId exists in local IndexedDB;
- every suggestion.photoId exists in that inspection.
```

Then it saves suggestions as annotations:

```text
source: "ai"
```

Internal AI metadata is stored in annotation attributes:

```text
_aiProvider
_aiConfidence
_aiReviewStatus
_aiImportedAt
_aiAcceptedAt
_aiOriginalSource
```

These fields are hidden from normal attribute preview in the annotation list.
